# Mon3tr - Телеграм бот для твоего времени.

**Mon3tr**, в соответствии с названием, будет не только **отслеживать** Ваши горящие **проекты** и срочные **дела**, но так же **оповещать** Вас о них. <br> _A может быть и о погоде!_

## Задача 1. Создание бота
Цель: Создать бота и добавить простейшие функции взаимодействия с ним
Пояснение:
1) Сделать заготовку чат-бота для Telegram, которая умеет отвечать на данные команды:
- `/about` — выводит краткую информацию про бота
- `/authors` — выводит информацию об авторах
- `/help` — выводит информацию обо всех доступных командах (можно запускать с аргументом: /help <command>)

## Задача 2. База для дедлайнов
**Цель:** Создать интерфейс для работы с базой данных (далее БД) и добавть базовые команды взаимодействия
**Пояснение:**
1. Развернуть БД
2. Реализовать класс-менеджер для взаимодействия БД и бота
3. Реализовать команды для взаимодействия пользователя с БД:
	- `/add` - добавление дедлайна в БД (Вызов: `/add <name> <date>`) <br>
		_Примечание: Не допускается создание дедлайна если не указано хоть одно из полей_ <br>

		**Форматы ввода:** <br>

		`<name>` - не пустая строка, не совпадающая с форматом даты <br>
		`<date>` - `dd.mm.yyyy` или `dd/mm/yyyy`, где <br>

		`dd` - натуральное число, соотвествующее номеру дня в месяце, не превышающее колличество дней в конкретном месяце <br>
		`mm` - натуральное число, соотвествующее номеру месяца, не превышает 12 <br>
		`yyyy` - натуральное число, соответствующее году <br>

		Допускается краткая запись дней, месяцев, лет: `d.m.yy` (и аналогичные).
		Не допускается пропуск одного или нескольких полей: `dd..yyyy` - запрещено!

		Номера дня и месяца урезаются до максимально доступного.
		Однозначное число года транслируется в `dd.mm.300y` - **имейте это в виду**.

	- ``` /mydeadlines``` - вывод списка всех дедлайнов пользователя (Вызов: ``` /mydeadlines```) <br> **Формат вывода:** ``` <id> <name> <date>``` (``` <id>``` - внутренний id в БД)

	- ``` /remove``` - удаление/закрытие дедлайна (Вызов: ``` /remove <id>```) <br> Пожалуйста не пытайтесь удалить дедлайны других пользователей

## Задача 3. Valley of voises (what?)
**Цель:** Внедрить поддержку асинхронного взаимодействия, ввести диалоговую систему для команд Задачи 2 и последующих задач
**Пояснение:**
1. Внедрить многопоточную работу бота
2. Доработать команды для работы с несколькими потоками
3. Расширить взаимодействие команд с пользователем до диалоговой системы
	- Запрос пропущеных/не введённых аргументов
	- Запрос некорректно введёных аргументов
4. Изменение `/remove`
	- Команда больше не будет требовать аргументов
	- При вызове команды пользователю будет выведен весь список его дедлайнов
	- После вывода списка бот запросит номер записи, которую необходимо удалить
	- После ответа пользователя данный дедлайн будет закрыт
	- Теперь номер дедлайна **не будет соответсвовать его id в базе данных!!!**

## Задача 4. This is fine
**Цель:** Ввести систему оповещения пользователя о горящих дедлайнах
**Пояснение:**
1. Добавление состояний записям дедлайнов: нормальный, горящий (близок срок), мертвый (дедлайн просрочен)
2. Внести правки в команды с выводом дедлайнов, для наглядного различия дедлайнов с разным состоянием
3. Добавить систему оповещений о горящих дедлайнах
4. Добавлена команда `/offset`:
	- Команда задает за сколько дней до крайнего срока дедлайн станет горящим (Вызов: ``` /offset <days>```) <br>
	**Формат ввода:** <br>
	- `<days>` - целое число не менее 0

## Задача 5. Seven nation army
**Цель:** Реализовать функции групп, групповых задач и добавить команды для взаимодействия с данным функционалом
**Пояснение:**
1. Реализовать команды базового взаимодействия с группамми:
	- `/create` - создать новую группу с заданным названием. <br>
	`/create <name>`
	- `/delete` - удалить выбранную группу. Работает только в диалоговом режиме.
	- `/invite` - создать пригласительный токен для выбранной группы. Только диалоговый режим.
	- `/join` - присоединиться к чужой группе по токену. <br>
	`/join <token>`
	- `/leave` - покинуть группу другого пользователя. Только диалоговый режим.
	- `/listGroups` - вывести все созданные группы.
	- `/membership` - вывести все группы, в которых состоит пользователь
2. Реализовать комманды взаимодействия задач и групп:
	- `/assign` - привязать выбранную задачу к выбранной группе, делая ее видимой для пользователей, состоящих в группе. Только диалоговый режим.
	- `/resign` - отвязать выбранную задачу от выбранной группы. Только диалоговый режим.
	- `/listAssignedTasks` - показать все задачи, привязанные к выбранной группе пользователя. Tолько диалоговый режим.
	- `/listMembershipTasks` - показать все задачи, привязанные к выбранной группе, в которой состоит пользователь. Только диалоговый режим.
3. Примечания:
	- Пользователь не состоит в группах, которые он создал.
	- Группы не являются обособленной от пользователя структурой, они лишь позволяют делиться своими задачами.
	- Пользователь сам решает какие задачи будут отображаться в той или иной группы.
